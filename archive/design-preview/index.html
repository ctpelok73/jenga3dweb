HUD · что закрыто из v0.2 фидбека

  • Player card усилена: цветной плинтус слева заменён на полноценный 54px glyph-блок (P1 / P2 / AI) — тяжёлым шрифтом, на цвете игрока. Имя ушло во вторичный текст. Теперь карточка визуально доминирует над Turns / Layer.
  • Три состояния таймера показаны стопкой справа (в реальной игре виден один): >10s обычный, 5–10s тонкая оранжевая рамка + тень того же цвета, ≤5s полная заливка + лёгкая пульсация (анимация hud-pulse). Предупреждение начинается на 5 секунд раньше.

  • Все эмодзи в HUD заменены на штриховые SVG из общей библиотеки (restart, target, volume).

settings · ключевые решения

  • Слайдер громкости — 2px рамка, оранжевая заливка, чёрный «нож»-knob. Visceral, читается с расстояния.
  • Группы опций (timer / difficulty) — единый компонент .set-opt, который повторно используем под все enum-настройки. В React это будет <OptionGroup/>.
  • Скины блоков теперь визуальные превью, не текстовые лейблы с эмодзи. Locked-skin приглушён + замочек.
  • Окружение (env theme) дорисую тем же паттерном — сейчас не показано чтобы экран не разрастался.

daily · ключевые решения

  • Задание дня — в строгой карточке, badge оранжевый, описание в Inter Tight. Состояние «выполнено» — зелёный stamp, читается мгновенно.

mobile · что зафиксировано до React

  • Нав-шапка → иконки без подписей (вариант .bar--m). Все 3 таба + mute помещаются в одну компактную строку.

06 confirm dialog · 2 variants

dialog · restart (destructive)

Начать заново?

Текущая игра будет потеряна. Это действие нельзя отменить.

dialog · back to menu

Выйти в меню?

Прогресс текущей игры будет утерян.

confirm · правила

  • Два размера риска. Destructive: красная рамка вокруг иконки, иконка alert, primary-кнопка с красной заливкой. Soft: чёрная рамка, иконка действия, тот же красный primary (т.к. результат всё равно — потеря прогресса).
  • Cancel всегда слева, привязан к ESC. Confirm справа, привязан к Enter. Hotkey-боксы видны.
  • Фон — diagonal hatched (45° штриховка) — это сразу читается как «модальное состояние», не путается с обычным экраном.
  • Под одну компонентную модель пойдут также: «Использовать Revive?», «Активировать код?», «Сменить тему?».
