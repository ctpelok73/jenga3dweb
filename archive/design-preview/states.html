01 / app phase top-level state machine — already in code as PHASE_*

App ├─ start // landing screen, mode select │ ├─ tutorial // first-run overlay (one-shot) │ └─ default ├─ playing // active game session │ ├─ idle // no block selected │ ├─ blockSelected // hi block + ghost slot visible │ ├─ executing // move animation in flight │ ├─ stabilizing // physics settling, no input │ ├─ aiThinking // AI computing move (mode 3 only) │ └─ paused // modal pause overlay open └─ gameOver // collapse or speed-timer expired ├─ default ├─ continued // after revive (continuedAfterCollapse=true) └─ replaying // replay player open
start → handleStart playing → collapse / time-up gameOver → restart playing ↻ backToMenu start

maps to react

  • Top-level phase already reduces to 'start' | 'playing' | 'gameOver' in useGameReducer. Substates of playing (idle / blockSelected / executing / stabilizing / aiThinking / paused) are currently derived from independent flags (selectedId, simulatingBlockIds, aiThinking, showPauseMenu). Worth a discriminated union later, not now — derivation works.

Unlocked

Achievement is in data.unlocked. Icon background fills accent. Meter hidden, date shown.

icon bg: accenticon color: inkcaption: "unlocked · 2026-05-12"

HUD chip variants

  • side (desktop) — vertical card on the right edge of the play field, ≥ 200px width.
  • inline (mobile, narrow) — horizontal chip docked above the action sheet.
  • Both forms share the same content model: { icon, name, current, target, percentage }. Component takes variant prop.

05 / dialog two severity levels

soft

Soft confirm

Action loses progress but is expected (Back to menu, Discard replay). Ink ring around icon, red primary stays for the destructive outcome.

icon ring: ink 3pxcancel key: ESCconfirm key: Enter
destructive

Destructive confirm

Irreversible-ish (Restart, Activate code that overwrites state). Warn ring around icon, alert glyph.

icon ring: warn 3pxicon: alertprimary: warn fill

existing destructive triggers

  • Restart in-game (PauseMenu / HUD) — destructive variant.
  • Back to main menu (PauseMenu / NavBar Menu tab while playing) — soft variant.
  • Reset settings (SettingsPanel footer) — soft variant.
  • Discard replay (future) — soft variant.

back-stack rule

  • NavBar tab clicks navigate; they don't stack. Going Settings → Achievements doesn't push, it replaces. The only stack is: caller → ConfirmDialog, which is one deep max.
  • ESC on a panel returns to caller (Start or Pause). ESC on Pause returns to game. ESC on game opens Pause. ESC on ConfirmDialog cancels.
